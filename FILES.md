# Структура файлов

Документ описывает назначение файлов в репозитории. История появления и
изменения этих файлов отслеживается git-коммитами и тегами, поэтому описание
ниже не привязано к конкретной версии программы.

## Документация

- `README.md` - описание программы, требований и работы с программой. Исходный
  текст перенесен из архивного `READ.ME` в Markdown-формат.
- `CHANGELOG.md` - история изменений в Markdown-формате. Заменяет архивные
  `HISTORY.TXT`; сами `HISTORY.TXT` в репозитории не хранятся.
- `FILES.md` - описание структуры репозитория и назначения файлов.

## Исходный код

- `ARITH.C` - основной файл программы. Содержит главный цикл, настройку
  параметров, генерацию примеров, проверку ответов, сохранение результатов и
  просмотр последнего протокола.
- `ASM.C` - низкоуровневые функции через BIOS/DOS-прерывания: клавиатура, дата,
  время, палитра, принтер и преобразование чисел.
- `ASM.H` - объявления функций из `ASM.C`.
- `CONIO.C` - собственные функции текстового вывода: видеостраницы, цвета,
  очистка экрана, позиционирование курсора, окна вывода и печать строк прямо в
  текстовый видеобуфер.
- `CONIO.H` - объявления функций и внешних переменных из `CONIO.C`.
- `MOUSE.C` - низкоуровневая работа с мышью через прерывание `33h`.
- `MOUSE.H` - объявления функций из `MOUSE.C`.
- `KEYS.H` - таблица и `#define`-константы кодов клавиатуры для обработки
  нажатий в DOS.

## Файлы сборки и запуска

- `ARITH.PRJ` - проектный файл Turbo C. Он связывает исходники `ARITH.C`,
  `CONIO.C`, `ASM.C`, `MOUSE.C` и используемые заголовки.
- `ARITH.EXE` - собранный DOS-исполняемый файл архивной программы.
- `ARITH.ICO` - иконка Windows для программы.
- `ARITH.PIF` - Windows PIF-файл для запуска DOS-программы.

## Данные программы и примеры

- `A.DAT` - сохраненные параметры программы: выбранные операции, имя, границы,
  количество примеров и сложность. В DOS имя файла нечувствительно к регистру;
  в исходном коде файл открывается как `a.dat`.
- `examples/sessions/` - архивные примеры пользовательских сеансов. Эти файлы
  нужны как образцы формата протокола для будущей поддержки и возможных тестов
  чтения результатов.
- `examples/sessions/*.000`, `examples/sessions/*.001` и т.п. - протоколы
  решений. Имя файла образовано из введенного имени пользователя, расширение
  выбирается в диапазоне `000`-`999`.

## Архивные файлы, которые не переносятся

В старом каталоге также есть файлы, которые не нужны как исходники или тестовые
данные репозитория:

- `*.OBJ`, `ARITH.SYM` - результаты сборки и отладочная информация.
- `ARITH.DSK`, `TCDEF.DSK`, `TCDEF.DPR`, `TCCONFIG.TC`, `status.me` - настройки
  и служебное состояние среды Turbo C / Turbo Debugger.
- `READ.ME` - перенесен в `README.md`.
- `HISTORY.TXT` - переносится в `CHANGELOG.md` при добавлении соответствующей
  версии.
- Каталоги `V100`, `V101`, `V102` в старом архиве - источники для
  последовательного переноса версий в git-историю.
- Каталоги `Arith`, `Arithmet`, `ddd` - дополнительные архивные копии,
  пользовательские данные или параллельные наборы файлов. Их содержимое не
  входит в основной проектный файл `ARITH.PRJ`.
